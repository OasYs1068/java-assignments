SL275: Module8: Exceptions

Exercise #1: Handle an Exception

(Level 1)

Objective

In this exercise you will use the try-catch block to handle a simple runtime exception.

Directions

The TestExceptions program prints out all of the command-line arguments, but it has a fatal flaw: it will eventually attempt to access beyond the args array bounds. 

public class TestExceptions {
  public static void main(String[] args) {
    for ( int i = 0; true; i++ ) {
      System.out.println("args[" + i + "] is '" + args[i] + "'");
    }
  }
}

Note: The code in this program is poorly implemented, because the loop should test for the exact size of the args array. However, the purpose of this exercise is to give you hands on experience with exceptions and how to catch them.

  1. Compile and run the TestExceptions program. The following is a sample of the output: 
    javac TestExceptions.java

java TestExceptions one two three
args[0] is 'one'
args[1] is 'two'
args[2] is 'three'
Exception in thread "main" java.lang.ArrayIndexOutOfBoundsException: 3
	at TestExceptions.main(TestExceptions.java:4)

  2. Modify the TestExceptions program to handle the ArrayIndexOutOfBoundsException by wrapping the for loop in a try block.

  3. Write a catch block that catches that exception and prints out the exception object and then prints that the application is quitting.

  4. Re-compile and run the program. The following is a sample of the output: 
    javac TestExceptions.java

java2 TestExceptions one two three
args[0] is 'one'
args[1] is 'two'
args[2] is 'three'
Exception caught: java.lang.ArrayIndexOutOfBoundsException: 3
Quiting...
