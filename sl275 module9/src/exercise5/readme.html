SL275: Module9: Text-Based Applications

Exercise #5: Document the Customer Class

(Level 1)

Objective

In this exercise you will use javadoc comment tags to document the Customer class. You will then generate the API documentation for the entire banking package.

Directions

Start by changing your working directory to SL275/mod09/exercise5 on your computer. Copy the previous Banking project files.

  1. Comment the declarations of the Customer class using a variety of javadoc tags. Make sure you use the @param and @returns tags on the appropriate declarations.

  2. Run the javadoc tool over the complete set of classes to generate the public API documentation as follows: 
    javadoc -d doc/api/public banking.domain banking.reports

    When you have done this, click here to view the generated documentation.

  3. Run the javadoc tool over the complete set of classes to generate the private API documentation as follows: 
    javadoc -private -d doc/api/private banking.domain banking.reports

    When you have done this, click here to view the generated documentation.
